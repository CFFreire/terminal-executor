crear un extension de vs code que permita ejecutar comando desde un archivo que tiene la extension .terminal

la idea es crear un archivo con la siguiente estructura de ejemplo, que aparezca arriba de cada descripcion es decir de cada ### un boton o un testo que diga ejecutar y al hacer clic se ejecute es comando de abajo

## Notes
- Commands are executed in a dedicated terminal named "Terminal Executor"
- The extension reuses the same terminal for all commands, keeping your terminal tabs clean
- If the terminal is closed, a new one will be created automatically on the next command execution
- Commands run from the title line (###) to either the next title or the end of the file

Informacion a considerar
    ## Status Indicators

    The extension provides visual feedback for command execution:

    - **▶ Execute** - Click to run the command
    - **⟳ Executing...** - Command is currently running
    - **✓ Success** - Command executed successfully
    - **✗ Error** - Command execution failed

    A timestamp showing how long ago the command was executed is also displayed.

    ## Implementation Details

    - Commands are executed in a dedicated terminal named "Terminal Executor"
    - The extension reuses the same terminal for all commands, keeping your terminal tabs clean
    - If the terminal is closed, a new one will be created automatically on the next command execution
    - The extension uses CodeLens to provide the execution buttons and status indicators
    - The regex pattern captures commands correctly even with complex syntaxes and special characters

    ## Requirements

    - Visual Studio Code v1.60.0 or higher

    ## Extension Settings

    This extension currently doesn't have any configurable settings.

    ## Known Issues

    - Success/error detection is currently time-based and doesn't reflect the actual command result
    - Terminal output is not captured for analysis

    ## Future Enhancements

    Potential future enhancements may include:
    - More accurate success/error detection based on exit codes
    - Command variables and substitutions
    - Command history tracking
    - Terminal session management
    - Custom terminal profiles
    - Command grouping and organization
